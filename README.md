Презентація проекту: PACMAN vs GHOST: The Duel
PACMAN vs GHOST: The Duel — це динамічна аркадна гра, розроблена на мові Python з використанням бібліотеки Pygame. Основна ідея проекту полягає в переосмисленні класичної механіки «хованок» у лабіринті, перетворюючи її на захопливе протистояння двох реальних гравців та автономного штучного інтелекту.

Ігровий процес та механіки:
У центрі гри — конфлікт двох сторін. Перший гравець керує Пакменом (жовте коло), чия мета — очистити лабіринт від усіх точок. Другий гравець бере на себе роль Червоного Привида, завдання якого — перехопити Пакмена до того, як він збере всі бонуси. Керування розділене: Пакмен використовує стрілочки, а Привид — клавіші WASD.

Для підвищення складності в гру додано третього учасника — Зеленого Привида-бота. Він рухається лабіринтом автономно, обираючи випадкові напрямки на кожному перехресті. Це створює елемент хаосу, змушуючи обох гравців постійно адаптуватися до непередбачуваної загрози.

Технічні особливості:
Проект реалізований з використанням матричної структури карти, де стіни та порожні проходи визначені цифровим кодом. Це дозволило створити точну систему колізій: персонажі не просто рухаються екраном, а взаємодіють із геометрією рівня. Використання векторної математики (pg.Vector2) забезпечує плавне переміщення та точне обчислення відстані для фіксації зіткнень.

Важливою частиною архітектури є система станів (Game States). Програма чітко розрізняє процес гри та фінальні екрани. Коли умови перемоги або поразки виконані, гра зупиняється і виводить відповідний текст («ПЕРЕМОГА!» або «ГРА ЗАКІНЧЕНА!») безпосередньо у вікно програми. Завдяки вбудованому меню перезапуску, гравець може миттєво почати новий раунд натисканням клавіші «Пробіл», що робить ігровий цикл безперервним.

Висновок та перспективи:
Цей проект демонструє можливості об’єктно-орієнтованого підходу в розробці ігор та логіки штучного інтелекту. У майбутньому планується розширення функціоналу: додавання супер-бонусів для Пакмена, створення складніших алгоритмів пошуку шляху для ботів та впровадження системи рівнів з різними лабіринтами.

Для перевірки стін використовується формула: MAP[int(y // TILE_SIZE)][int(x // TILE_SIZE)].

Як це працює: Координати гравця в пікселях діляться на розмір клітинки (40 пікселів). Це конвертує фізичне положення у «номер рядка та стовпця» в матриці MAP. Якщо в цій комірці стоїть 1, рух блокується. Це найнадійніший спосіб створення лабіринтів.
